# ตั้งค่าการฝากเงิน

## คำอธิบาย
หน้านี้ใช้สำหรับจัดการ บัญชีธนาคารออโต้ ที่เชื่อมต่อกับระบบหลังบ้าน เช่น บัญชีที่ใช้ตรวจสอบยอดฝาก-ถอนอัตโนมัติ สถานะการเชื่อมต่อ และการเปิด-ปิดการใช้งานบัญชี ผู้ดูแลระบบต้องได้รับสิทธิ์ด้านจัดการบัญชีธนาคาร จึงจะสามารถเข้าถึงและใช้งานหน้านี้ได้

## สิทธิ์การเข้าใช้งาน
หน้านี้ใช้สำหรับจัดการการตั้งค่าที่เกี่ยวข้องกับระบบฝาก และธุรกรรมภายในระบบ ผู้ดูแลระบบต้องได้รับสิทธิ์ด้านการตั้งค่าการฝากเงินจึงจะสามารถเข้าถึงและใช้งานหน้านี้ได้
*  **สิทธิ์ระดับดูข้อมูล:** สามารถเข้าดูรายการบัญชีธนาคาร รายละเอียดบัญชี และสถานะการใช้งานได้ แต่ไม่สามารถเพิ่ม แก้ไข ลบ หรือบันทึกการเปลี่ยนแปลงข้อมูลได้
*  **สิทธิ์ระดับเเก้ไข:** สามารถเพิ่ม แก้ไข ลบ เปิด-ปิดการใช้งาน และจัดการข้อมูลบัญชีธนาคารออโต้หลังบ้านได้

## วิธีเข้าถึงหน้านี้
* ไปที่: **เมนูหลังบ้าน** > หมวด **ช่องทางการชำระเงิน** > เลือกเมนูย่อย **ธนาคารออโต้**

## รายละเอียดของแต่ละส่วนในหน้าจอ

### 1. ส่วนการตั้งค่าด่วนและหมวดหมู่หลัก

<a href="/images/auto-banks/image1.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/auto-banks/image1.png" alt="หน้าต่าง ส่วนการตั้งค่าด่วนและหมวดหมู่หลัก" style="width: 100%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

ส่วนนี้อยู่ด้านบนสุด ใช้ควบคุมทิศทางการทำงานโดยรวมของระบบการชำระเงิน

* **ปุ่ม อัปเดตยอดคงเหลือ:** (มุมขวาบน) คลิกเพื่อยิงคำสั่งไปดึงข้อมูลยอดเงินสดล่าสุดจากผู้ให้บริการทุกรายมาแสดง
* **ปุ่ม + เพิ่มบัญชี / + สร้างช่วงปิดปรับปรุง:** ปุ่มดำเนินการหลัก (ข้อความบนปุ่มจะเปลี่ยนไปตามแท็บที่เลือกใช้งานอยู่)
* **การตั้งค่าด่วน:**
  * **สวิตช์ ฝากผ่านบัญชีธนาคาร:** เปิดเพื่อสลับไปใช้หน้าบัญชีธนาคารปกติแทนระบบธนาคารออโต้
  * **สวิตช์ ให้สมาชิกเลือกช่องทางฝาก:** หากปิด ระบบจะวิเคราะห์และเลือกช่องทางฝากที่ดีที่สุดให้สมาชิกอัตโนมัติ
  * **วิธีอนุมัติถอน:** กำหนดให้การโอนเงินออกเป็นการกระทำโดย "แอดมินตรวจเอง" หรือ "ธนาคารออโต้โอนให้"
* **แท็บเมนูหลัก:** แบ่งการทำงานออกเป็น 2 หน้าหลักคือ **บัญชี** และ **ช่วงปิดปรับปรุง**

### 2. หมวดบัญชีเเละส่วนตัวกรอง

<a href="/images/auto-banks/image2.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/auto-banks/image2.png" alt="หมวดบัญชีเเละส่วนตัวกรอง" style="width: 100%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

ส่วนนี้ใช้สำหรับสลับหน้าจอจัดการบัญชีผู้ให้บริการตามประเภทการทำธุรกรรม และใช้ตัวกรองเพื่อค้นหาบัญชีที่ต้องการตรวจสอบสถานะได้อย่างรวดเร็ว

* **แท็บหมวดบัญชีย่อย:** คลิกเพื่อเปลี่ยนหน้าตารางข้อมูล โดยแบ่งออกเป็น 3 หมวด ได้แก่
  * **ฝากผ่าน QR:** บัญชีที่ใช้รับเงินผ่านการสแกนคิวอาร์โค้ด (PromptPay / QR Payment)
  * **ฝากผ่านธนาคาร:** บัญชีที่ใช้รับเงินผ่านการโอนเข้าเลขบัญชีธนาคาร
  * **ถอน:** บัญชีที่ใช้สำหรับโอนเงินออกจากระบบไปให้สมาชิก
* **ตัวกรองบัญชี:**
  * **ค้นหา:** ช่องสำหรับพิมพ์เพื่อค้นหาจาก **ชื่อบัญชี**
  * **สถานะบัญชี:** กรองดูเฉพาะบัญชีที่กำลังเปิดใช้งาน หรือบัญชีที่ปิดใช้งานอยู่
  * **การเชื่อมต่อ:** ตรวจสอบสถานะการสื่อสารของ API ว่า **ปกติ** หรือ **ล้มเหลว**
  * **ซ่อมบำรุง:** กรองดูเฉพาะบัญชีที่กำลังอยู่ในช่วงเวลาปิดปรับปรุงระบบ
  * **บัญชีหลัก:** (ตัวกรองนี้จะแสดงขึ้นมาเฉพาะเมื่อคลิกที่แท็บ **ถอน**) ใช้สำหรับกรองหาบัญชีที่ถูกตั้งให้เป็นกระเป๋าจ่ายเงินหลักของระบบ


### 3. ตารางข้อมูลบัญชี

<a href="/images/auto-banks/image3.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/auto-banks/image3.png" alt="ตารางข้อมูลบัญชี" style="width: 100%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

ตารางนี้แสดงรายชื่อบัญชีผู้ให้บริการทั้งหมด ใช้สำหรับตรวจสอบสถานะ ยอดเงิน และจัดการธุรกรรม (รูปแบบตารางจะเหมือนกันทั้งหมวด **ฝากผ่าน QR**, **ฝากผ่านธนาคาร** และ **ถอน**)

**ข้อมูลทั่วไปในตาราง**
* **คอลัมน์ ชื่อบัญชี / ผู้ให้บริการ:** แสดงชื่อบัญชีที่แอดมินตั้งไว้ และชื่อของระบบผู้ให้บริการ (เช่น SMILEPAYZ)
* **คอลัมน์ ซ่อมบำรุง:** แสดงสถานะการทำงานของ API ในปัจจุบัน ว่าเชื่อมต่อ **ปกติ** หรือกำลังอยู่ในช่วงปิดปรับปรุง
* **คอลัมน์ ยอดเงินคงเหลือ:** ยอดเงินจริงของแพลตฟอร์มเรา ที่ฝากค้างอยู่ในระบบของผู้ให้บริการรายนั้น ๆ
* **คอลัมน์ ลิงก์:** ปุ่มทางลัดสำหรับกดข้ามไปยังหน้าเว็บไซต์หลังบ้านของผู้ให้บริการนั้น ๆ โดยตรง เพื่อความสะดวกรวดเร็ว
* **คอลัมน์ เงินฝากขั้นต่ำ / สูงสุด:** จำนวนเงินน้อยสุดและมากสุดที่อนุญาตให้ทำรายการได้ใน 1 ครั้งผ่านช่องทางนี้
* **คอลัมน์ สถานะ:** (แสดงเฉพาะในหมวดฝากเงิน) สวิตช์สำหรับเปิด-ปิดการแสดงผล หากเปิดสวิตช์ (สีขาว) สมาชิกจะสามารถมองเห็นและทำรายการผ่านช่องทางนี้ได้

**ความหมายของ "ปุ่มจัดการ"**
<a href="/images/auto-banks/image4.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/auto-banks/image4.png" alt="ตารางข้อมูลบัญชี" style="width: 40%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

* **ปุ่ม ฝาก:** ใช้เมื่อแอดมินโอนเงินจริงเติมเข้าไปในกระเป๋าของผู้ให้บริการ แล้วมากดปุ่มนี้เพื่อบันทึกยอดเงินเข้าสู่ระบบหลังบ้าน
* **ปุ่ม ถอน:** ใช้เมื่อต้องการดึงเงินออกจากผู้ให้บริการ โดยระบบจะส่งคำสั่งผ่าน API ให้ผู้ให้บริการโอนยอดเงินที่ค้างอยู่ กลับมาเข้าบัญชีธนาคารของแพลตฟอร์ม
* **ปุ่ม แก้ไข / ลบ:** ใช้สำหรับเปลี่ยนรหัส API Keys ปรับลิมิตยอดเงิน หรือลบการเชื่อมต่อบัญชีนี้ทิ้ง

### 4. ช่วงปิดปรับปรุง

<a href="/images/auto-banks/image5.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/auto-banks/image5.png" alt="แท็บช่วงปิดปรับปรุง" style="width: 100%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

ใช้สำหรับดูประวัติและตั้งเวลาให้ระบบระงับการทำธุรกรรมผ่านธนาคารออโต้ชั่วคราว เพื่อป้องกันรายการค้างเมื่อธนาคารปลายทางปิดปรับปรุง

* **ตารางข้อมูล:** แสดงรายชื่อบัญชีที่มีกำหนดการซ่อมบำรุง, เวลาเริ่มต้น, เวลาสิ้นสุด และสถานะการทำงาน
* **ปุ่ม + สร้างช่วงปิดปรับปรุง:** (มุมขวาบน) ใช้เพิ่มกำหนดการใหม่

หน้าต่างสำหรับการ **เพิ่มบัญชี** และ **แก้ไข** จะมีหน้าตาและช่องกรอกข้อมูลเหมือนกันทุกประการ ใช้จัดการความปลอดภัย (API Keys)
*เมื่อกดเพิ่มบัญชี ระบบจะให้คลิก **เลือกผู้ให้บริการ** (เช่น SMILEPAYZ) ก่อนเข้าสู่หน้านี้*

* **ข้อมูลทั่วไป:** ระบุ **ชื่อบัญชี**, **ฝากขั้นต่ำ**, **ฝากสูงสุด** และสวิตช์ **เปิดใช้งาน** บัญชี
* **ข้อมูลรับรองการเชื่อมต่อ (API Keys):** ต้องกรอกให้ตรงตามที่ผู้ให้บริการกำหนดทุกตัวอักษร
  * **รหัสร้านค้า (MERCHANT ID)** และ **MERCHANT SECRET (HEX)**
  * **MERCHANT PRIVATE KEY / PUBLIC KEY:** แอดมินสามารถคลิกปุ่ม **อัปโหลดไฟล์** (`.pem`, `.key`, `.txt`, `.json`) เพื่อดึงคีย์เข้าระบบได้อย่างแม่นยำ ป้องกันการคัดลอกข้อความผิดพลาด
  * **เลขบัญชีรับเงินปลายทาง (SETTLEMENT):** เลขบัญชีหลักของแพลตฟอร์มสำหรับรับยอดโอนคืน
* **ปุ่ม บันทึก:** กดยืนยันเพื่อบันทึกและให้ระบบเริ่มทดสอบการเชื่อมต่อ

### 6. หน้าต่าง สร้างช่วงปิดปรับปรุง

<a href="/images/auto-banks/image7.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/auto-banks/image7.png" alt="หน้าต่าง สร้างช่วงปิดปรับปรุง" style="width: 60%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

เปิดจากแท็บ "ช่วงปิดปรับปรุง" ใช้สำหรับตั้งเวลาพักการใช้งานของระบบ API ชั่วคราว

* **บัญชี:** เลือกรายชื่อบัญชีธนาคารออโต้ที่ต้องการระงับการทำงาน
* **ประเภท:** เลือกได้ว่าต้องการระงับการ **ฝาก**, **ถอน** หรือ **ทั้งหมด (ฝาก + ถอน)**
* **เริ่มต้น:** กำหนดวันที่และเวลาที่จะเริ่มปิดระบบ
* **สิ้นสุด:** กำหนดวันที่และเวลาที่จะให้ระบบเปิดกลับมาอัตโนมัติ (หากปล่อยว่างไว้ ระบบจะปิดบัญชีนี้ไปเรื่อย ๆ จนกว่าแอดมินจะมากดยกเลิกเอง)
* **เหตุผล:** บันทึกข้อความหมายเหตุ เช่น "SCB แจ้งปิดปรับปรุงระบบเที่ยงคืน"
* **ปุ่ม สร้าง:** ยืนยันการบันทึกตารางเวลา

* **หน้าต่างถอนเงิน:** เป็นการยิงคำสั่ง API ระบุ **จำนวนเงิน**, **ธนาคาร/เลขบัญชี/ชื่อบัญชีปลายทาง** (ของแพลตฟอร์มเรา) 
  * **ข้อควรระวัง:** มีคำเตือนสีเหลืองแจ้งว่า ต้องรอผู้ให้บริการระบบปลายทางประมวลผลและยืนยันสถานะกลับมาก่อน รายการจึงจะเสร็จสมบูรณ์

## ปัญหาที่พบบ่อย (FAQ)

**1. ทำไมถึงเปิดสวิตช์ "บัญชีหลัก" ในแท็บถอนพร้อมกันหลายบัญชีไม่ได้ ?**
* **สาเหตุ:** เป็นมาตรการป้องกันความผิดพลาดและซ้ำซ้อนของการกระจายยอดเงิน ระบบจึงบังคับให้มี "บัญชีถอนหลัก" ในการทำหน้าที่โอนเงินออกให้สมาชิกเพียงช่องทางเดียวในแต่ละช่วงเวลา
* **วิธีแก้ไข:** หากต้องการสลับไปใช้บัญชีของค่ายอื่น ให้คลิกเปิดสวิตช์ในคอลัมน์ **บัญชีหลัก** ที่แถวของบัญชีใหม่ได้เลย ระบบจะปิดการทำงานของบัญชีเก่าให้โดยอัตโนมัติ

**2. ตั้ง "ช่วงปิดปรับปรุง" โดยไม่ใส่วันสิ้นสุดได้หรือไม่ ?**
* **สาเหตุ:** ผู้ให้บริการแจ้งปิดฉุกเฉินโดยไม่ระบุเวลาเปิดที่ชัดเจน
* **วิธีแก้ไข:** สามารถทำได้ หากแอดมินปล่อยช่อง **สิ้นสุด (ว่าง = จนกว่าจะยกเลิก)** เอาไว้ ระบบจะระงับการใช้งานบัญชีนั้นไปอย่างไม่มีกำหนด จนกว่าแอดมินจะเข้ามาแก้ไขตารางหรือกดลบช่วงเวลาปิดปรับปรุงนั้นทิ้งด้วยตนเอง

**3. เชื่อมต่อ API แล้ว แต่สถานะแสดงเป็น "ล้มเหลว" เกิดจากอะไร ?**
* **สาเหตุ:** การคัดลอกข้อความ API Key มาวางอาจมีอักขระเว้นวรรคส่วนเกินติดมาด้วย หรือ IP Address ของเซิร์ฟเวอร์แพลตฟอร์มยังไม่ได้ถูกเพิ่มเข้าระบบรักษาความปลอดภัย (Whitelist) ฝั่งผู้ให้บริการ
* **วิธีแก้ไข:** เข้าไปที่ปุ่ม **แก้ไข** แนะนำให้ลบข้อมูลคีย์เดิมออกและใช้วิธีคลิก **อัปโหลดไฟล์** `.pem` หรือ `.txt` แทน หากข้อมูลคีย์ถูกต้องทั้งหมดแล้ว ให้ติดต่อฝ่ายบริการลูกค้าของ Payment Gateway เพื่อขอตรวจสอบ IP Whitelist

**4. กดทำคำสั่งถอนเงินจากบัญชีออโต้แล้ว ทำไมยอดเงินยังไม่เข้าบัญชีธนาคารแพลตฟอร์ม ?**
* **สาเหตุ:** การถอนเงินกลับจากระบบ Payment Gateway มีระยะเวลาประมวลผลตามคิว (SLA) ของผู้ให้บริการรายนั้น ๆ ซึ่งแตกต่างจากการโอนเงินผ่านแอปธนาคารทั่วไปที่เข้าทันที
* **วิธีแก้ไข:** ตามที่มีข้อความแจ้งเตือนสีเหลืองในหน้าต่าง แอดมินต้องรอระยะเวลาหนึ่งเพื่อให้ระบบปลายทางโอนเงินสำเร็จ หากล่าช้าผิดปกติให้แคปหน้าจอที่มีเลขอ้างอิงนำไปตรวจสอบกับผู้ให้บริการ

**5. ทำไมหน้าต่างแก้ไขบัญชี ถึงหน้าตาเหมือนตอนเพิ่มบัญชีใหม่ทุกอย่าง ต้องกรอกใหม่ทั้งหมดหรือไม่ ?**
* **สาเหตุ:** เป็นการออกแบบระบบให้แอดมินจัดการฟิลด์ข้อมูลได้ครบถ้วนในจุดเดียว (UI Consistency)
* **วิธีแก้ไข:** ข้อมูลเดิมที่เคยบันทึกไว้จะถูกดึงมาแสดงอยู่แล้ว แอดมินไม่จำเป็นต้องกรอกใหม่ทั้งหมด เพียงแค่แก้ไขเฉพาะช่องที่ต้องการอัปเดต (เช่น ต้องการเปลี่ยนลิมิต **เงินฝากสูงสุด**) แล้วกด **บันทึกการเปลี่ยนแปลง** ได้เลย
