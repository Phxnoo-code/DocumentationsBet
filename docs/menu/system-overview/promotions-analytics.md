# วิเคราะห์โปรโมชั่น
## คำอธิบาย

หน้าสำหรับติดตามและประเมินผลลัพธ์การจัดกิจกรรมการตลาด โดยระบบจะรวบรวมข้อมูลการกดรับโปรโมชั่น สถิติการเล่นผ่านเทิร์น ยอดเงินฝาก และตัวเลขกำไรขาดทุนของแต่ละแคมเปญในช่วงวันที่เลือก เพื่อให้เจ้าของเว็บมองเห็นประสิทธิภาพความคุ้มค่าสูงสุด

## สิทธิ์การเข้าใช้งาน
* แอดมินต้องได้รับสิทธิ์ในหมวด **จัดการการตลาด** จึงจะสามารถเข้าถึงหน้าจอและมองเห็นตัวเลขสถิติของโปรโมชั่นในพาร์ทนี้ได้

## วิธีเข้าถึงหน้านี้
* **ไปที่:** **เมนูหลังบ้าน** > หมวด **สรุปภาพรวม** > เลือกเมนูย่อย **วิเคราะห์โปรโมชั่น**

## รายละเอียดของแต่ละส่วนในหน้าจอ

### 1. สรุปภาพรวมโปรโมชั่น
กล่องสรุปข้อมูลหน้าบอร์ดแถวบนสุด ช่วยให้เห็นตัวเลขภาพรวมของทุกกิจกรรมอย่างรวดเร็ว:
<a href="/images/promotions-analytics/image-5.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/promotions-analytics/image-5.png" alt="การ์ดสรุปภาพรวมโปรโมชั่น" style="width: 100%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>
กล่องสรุปข้อมูลแถวบนสุด ใช้สำหรับแสดงภาพรวมของผลลัพธ์จากโปรโมชั่นในช่วงเวลาที่เลือก โดยมีการ์ดสรุปทั้งหมด 4 ใบ (ไม่สามารถคลิกดูรายละเอียดเพิ่มเติมได้)

**1.1 โปรโมชั่นในหน้านี้**
แสดงจำนวนประเภทของโปรโมชั่นที่ถูกนำมาคำนวณในรายงาน เพื่อให้ทราบว่าข้อมูลด้านล่างครอบคลุมโปรโมชั่นทั้งหมดกี่รายการ

**1.2 สมาชิกไม่ซ้ำ**
แสดงจำนวนครั้งที่มีการรับโปรโมชั่นทั้งหมด พร้อมแยกสถานะเป็น

**1.3 จำนวนรับทั้งหมด**
* **ผ่าน** สมาชิกทำยอดเทิร์นครบตามเงื่อนไข
* **ไม่ผ่าน** สมาชิกทำยอดไม่ถึงเงื่อนไข หรือรายการถูกยกเลิก

**1.4 กำไรรวม**
แสดงผลตอบแทนของโปรโมชั่นในรูปแบบตัวเลขทางการเงิน โดยคำนวณจากยอดฝากที่เกี่ยวข้องกับโปรโมชั่น พร้อมแสดงยอดฝากรวมเพื่อใช้ประเมินความคุ้มค่าของแคมเปญ


### 2. ตัวกรองข้อมูล
<a href="/images/promotions-analytics/image-3.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/promotions-analytics/image-3.png" alt="ตัวกรองข้อมูล " style="width: 100%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

ส่วนนี้ใช้สำหรับค้นหาและคัดกรองข้อมูลโปรโมชั่น เพื่อช่วยให้แอดมินวิเคราะห์แคมเปญได้อย่างแม่นยำ โดยข้อมูลจะอ้างอิงจากช่วงเวลาที่มีการรับโปรโมชั่นและการทำเทิร์นจริง ประกอบด้วยเครื่องมือ 3 ส่วน ดังนี้

**2.1 ค้นหาโปรโมชั่น**
ค้นหาด้วย ชื่อโปรโมชั่น เหมาะสำหรับกรณีที่มีหลายแคมเปญและต้องการดูข้อมูลเฉพาะรายการ

**2.2 ช่วงวันที่**
กำหนดช่วงเวลาที่ต้องการดูสถิติ พร้อมปุ่มลัดสำหรับเลือกช่วงเวลายอดนิยม ได้แก่
* **ทั้งหมด** แสดงข้อมูลตั้งแต่เริ่มแคมเปญจนถึงปัจจุบัน
* **30 วันล่าสุด** แสดงข้อมูลย้อนหลัง 30 วัน

**2.3 เรียงข้อมูล**
ใช้จัดลำดับโปรโมชั่นในตาราง เพื่อช่วยค้นหาแคมเปญที่มีผลงานโดดเด่นได้รวดเร็ว โดยมีตัวเลือกดังนี้
* **กำไรมากสุด** เรียงตามกำไรสูงสุด
* **ยอดฝากมากสุด** เรียงตามยอดฝากสูงสุด
* **จำนวนรับมากสุด** เรียงตามจำนวนครั้งที่มีการรับโปรโมชั่น
* **สมาชิกมากสุด** เรียงตามจำนวนสมาชิกที่เข้าร่วมมากที่สุด
* **ไม่ผ่านเทิร์นมากสุด** เรียงตามจำนวนรายการที่ทำเทิร์นไม่สำเร็จ เหมาะสำหรับใช้วิเคราะห์และปรับปรุงเงื่อนไขโปรโมชั่น


### 3. ตารางข้อมูลรายงาน

* **ภาพตัวอย่างการแสดงผลในตารางหลังบ้าน:**

<a href="/images/promotions-analytics/image-7.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/promotions-analytics/image-7.png" alt="ภาพตัวอย่างการแสดงผลในตารางหลังบ้าน" style="width: 100%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

ส่วนนี้คือตารางสรุปผลลัพธ์ของโปรโมชั่นทั้งหมดตามเงื่อนไขที่เลือก ช่วยให้แอดมินเปรียบเทียบประสิทธิภาพและวิเคราะห์ความคุ้มค่าของแต่ละแคมเปญได้ โดยมีข้อมูลสำคัญดังนี้

* **มุมมองการแสดงผล** เลือกดูข้อมูลได้ทั้งแบบ ตาราง หรือ การ์ด เพื่อให้เหมาะกับการใช้งานบนแต่ละอุปกรณ์
* **การเลื่อนดูข้อมูล** สามารถเลื่อนแถบ Scroll ด้านล่างเพื่อดูคอลัมน์เพิ่มเติมได้
* **ชื่อโปรโมชั่น** ชื่อของแคมเปญ
* **ประเภท** ระบุประเภทของผู้รับโปรโมชั่น เช่น สมาชิกทั่วไป หรือสมาชิกใหม่
* **จำนวนสมาชิก** จำนวนผู้รับโปรโมชั่นแบบไม่ซ้ำ
* **จำนวนรับทั้งหมด / สถานะเทิร์น**จำนวนครั้งที่มีการรับโปรโมชั่น พร้อมแยกจำนวนที่ทำเทิร์นผ่านและไม่ผ่าน
* **กำไร / เฉลี่ย (บาท)** กำไรรวมของโปรโมชั่น และกำไรเฉลี่ยต่อการรับ 1 ครั้ง
* **ยอดฝากรวม / เฉลี่ย (บาท)** จำนวนเงินฝากทั้งหมดที่เข้ามาพร้อมกับการรับโปรนี้ พร้อมยอดฝากเฉลี่ยต่อบิล
* **ยอดคงเหลือหลังทำเทิร์นสำเร็จ / เฉลี่ย (บาท)** ยอดฝากทั้งหมดและยอดฝากเฉลี่ยต่อบิล
* **ยอดโยกเครดิตทั้งหมด / เฉลี่ย (บาท)** จำนวนเครดิตที่ถูกโยกกลับเข้ากระเป๋าหลัก
* **ยอดเทิร์นที่ทำได้ในกรณีทำเทิร์นไม่สำเร็จ / เฉลี่ย** แสดงยอดเทิร์นเฉลี่ยของกลุ่มที่ทำเทิร์นไม่ผ่าน เพื่อใช้วิเคราะห์ว่าเงื่อนไขอาจยากเกินไปหรือไม่

> **หมายเหตุ:**
> หากต้องการดูรายละเอียดของโปรโมชั่นใดเพิ่มเติม สามารถคลิกที่แถวของโปรโมชั่นนั้น หรือกดลูกศรด้านขวาสุดเพื่อเข้าสู่หน้าเจาะลึกได้

#### 3.1 หน้าดูรายละเอียด

<a href="/images/promotions-analytics/image-8.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/promotions-analytics/image-8.png" alt="หน้าดูรายละเอียด" style="width: 100%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

เมื่อคลิกเลือกโปรโมชั่นจากตาราง ระบบจะแสดงรายละเอียดของสมาชิกที่รับโปรโมชั่นนั้น พร้อมข้อมูลสำหรับวิเคราะห์ผลลัพธ์ของแคมเปญ โดยมีองค์ประกอบดังนี้

* **ข้อมูลพื้นฐานโปรโมชั่น** แสดงชื่อโปรโมชั่น ประเภท สถานะ และช่วงเวลาของข้อมูลที่กำลังดู
* **ปุ่มรีเฟรช** ใช้อัปเดตข้อมูลให้เป็นปัจจุบัน
* **สมาชิกในหน้านี้** จำนวนสมาชิกที่แสดงอยู่ในตาราง
* **จำนวนรับทั้งหมด** จำนวนครั้งที่มีการรับโปรโมชั่น พร้อมแยกสถานะผ่านและไม่ผ่าน
* **ยอดฝากรวม** ยอดฝากทั้งหมดที่เกิดจากโปรโมชั่นนี้
* **กำไรรวม** กำไรที่เกิดจากโปรโมชั่นนี้
* **ค้นหาสมาชิก** ค้นหาด้วย Username หรือเบอร์โทรศัพท์
* **ช่วงวันที่** ใช้เลือกช่วงเวลาที่ต้องการดูข้อมูล พร้อมปุ่มลัด ทั้งหมด และ 30 วันล่าสุด
* **ตารางข้อมูลสมาชิก** แสดงรายชื่อสมาชิก สถานะการทำเทิร์น ยอดฝาก และกำไรของแต่ละราย เพื่อใช้วิเคราะห์รายบุคคล

## ปัญหาที่พบบ่อย (FAQ)

**1. ทำไม "สมาชิกไม่ซ้ำ" น้อยกว่า "จำนวนรับทั้งหมด" ?**
* **สาเหตุ:** จำนวนรับทั้งหมดนับตามจำนวนครั้งที่มีการกดรับ ส่วนสมาชิกไม่ซ้ำนับตามจำนวนยูสเซอร์จริง
* **คำแนะนำ:** หากต้องการดูว่าสมาชิกรายใดรับโปรโมชั่นซ้ำบ่อย สามารถเปิดหน้าเจาะลึกเพื่อดูข้อมูลรายบุคคลได้

**2. ทำไมกำไรแสดงเป็นตัวเลขติดลบ ?**
* **สาเหตุ:** สมาชิกถอนเงินออกไปมากกว่ายอดฝากที่ใช้รับโปรโมชั่น ทำให้โปรโมชั่นขาดทุน
* **วิธีแก้ไข / คำแนะนำ:** อาจพิจารณาปรับเงื่อนไขเทิร์นโอเวอร์หรือเปอร์เซ็นต์โบนัสในแคมเปญถัดไป

**3. ค้นหาสมาชิกไม่พบ ทั้งที่เคยรับโปรโมชั่นนี้ ?**
* **สาเหตุ:** ช่วงวันที่ที่เลือกอาจไม่ครอบคลุมวันที่สมาชิกกดรับโปรโมชั่น
* **วิธีแก้ไข:** ขยายช่วงวันที่เป็น 30 วันล่าสุด หรือ ทั้งหมด แล้วกด รีเฟรช อีกครั้ง

**4. จะประเมินได้อย่างไรว่าเงื่อนไขเทิร์นโอเวอร์ยากเกินไปหรือไม่ ?**

  **ให้สังเกตคอลัมน์ ยอดเทิร์นที่ทำได้ในกรณีทำเทิร์นไม่สำเร็จ**
  * **ค่าเฉลี่ยต่ำมาก แสดงว่าสมาชิกส่วนใหญ่เครดิตหมดก่อนทำเทิร์นได้มากพอ**
  * **ค่าเฉลี่ยใกล้เป้าหมาย แสดงว่าเงื่อนไขอยู่ในระดับที่เหมาะสม**

**5. จะดูได้อย่างไรว่าโปรโมชั่นไหนควรทำต่อ ?**

  **ใช้ตัวเลือก เรียงข้อมูล เพื่อช่วยวิเคราะห์**
  * **กำไรมากสุด หรือ สมาชิกมากสุด เหมาะสำหรับหาโปรโมชั่นที่ทำผลงานดี**
  * **ไม่ผ่านเทิร์นมากสุด เหมาะสำหรับหาโปรโมชั่นที่ควรปรับปรุงหรือทบทวนเงื่อนไข**
