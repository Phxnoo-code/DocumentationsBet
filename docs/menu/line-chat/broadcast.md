# บรอดแคสต์ข้อความ

## คำอธิบาย

หน้านี้ใช้สำหรับส่งข้อความ Broadcast ไปยังหลายห้องแชท LINE พร้อมกัน แอดมินสามารถสร้างข้อความได้หลายรูปแบบ เช่น ข้อความ สติกเกอร์ รูปภาพ วิดีโอ เสียง ไฟล์ และ Flex Message โดยระบบจะแสดงคำเตือนก่อนส่ง เพื่อให้ตรวจสอบจำนวนผู้รับและตัวอย่างข้อความก่อนยืนยันการส่งจริง

## สิทธิ์การเข้าใช้งาน

การเข้าถึงหน้า **บรอดแคสต์ข้อความ** สามารถเข้าใช้งานได้ทั้งแอดมินที่มี **สิทธิ์ระดับดูข้อมูล** และ **สิทธิ์ระดับจัดการ** โดยระบบจะแบ่งความสามารถตามระดับสิทธิ์ เพื่อควบคุมความปลอดภัยของการส่งข้อความ LINE ดังนี้:

* **สิทธิ์ระดับดูข้อมูล:** แอดมินสามารถเข้าดูหน้าส่ง Broadcast รูปแบบข้อความ ตัวอย่างข้อความ และข้อมูลที่เกี่ยวข้องได้เท่านั้น แต่ไม่สามารถสร้าง แก้ไข ลบ หรือยืนยันส่ง Broadcast ได้
* **สิทธิ์ระดับจัดการ:** แอดมินสามารถสร้างข้อความ Broadcast เพิ่มข้อความหลายรายการ เลือกประเภทข้อความ ตรวจสอบตัวอย่าง ตรวจสอบจำนวนผู้รับ และยืนยันส่ง Broadcast ไปยังห้องแชท LINE ได้

## วิธีเข้าถึงหน้านี้

* **ไปที่:** **เมนูหลังบ้าน** > หมวด **แชท LINE** > เลือกเมนูย่อย **บรอดแคสต์ข้อความ**

## รายละเอียดของแต่ละส่วนในหน้าจอ

### 1. ส่วนหัวหน้าและคำเตือนก่อน Broadcast

<a href="/images/broadcast/image2.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/broadcast/image2.png" alt="ส่วนหัวหน้าและคำเตือนก่อน Broadcast" style="width: 100%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

ส่วนบนของหน้าจอแสดงชื่อหน้า **บรอดแคสต์ข้อความ** พร้อมคำอธิบายว่าใช้สำหรับส่งข้อความไปยังหลายห้องแชท LINE พร้อมกัน

* **บรอดแคสต์ข้อความ:** ชื่อหน้าสำหรับส่งข้อความ Broadcast ไปยังหลายห้องแชท LINE
* **คำอธิบายใต้ชื่อหน้า:** แจ้งว่าหน้านี้ใช้ส่งข้อความไปยังหลายห้องแชท LINE พร้อมกัน
* **คำเตือนก่อน Broadcast:** แจ้งให้แอดมินตรวจสอบข้อความและจำนวนผู้รับก่อนส่งจริง
* **ข้อความแจ้งเตือน:** ระบุว่า Broadcast จะส่งแบบ Push Message ไปยังห้องแชท LINE ที่เปิดอยู่ทั้งหมด และจำนวนผู้รับจริงอาจแตกต่างตามข้อจำกัดของ LINE Official Account

> ข้อควรระวัง: ควรตรวจสอบข้อความ จำนวนผู้รับ และความถูกต้องของเนื้อหาก่อนส่งทุกครั้ง เพราะ Broadcast เป็นการส่งข้อความไปยังหลายห้องพร้อมกัน

### 2. สร้างข้อความ Broadcast

<a href="/images/broadcast/image1.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/broadcast/image1.png" alt="สร้างข้อความ Broadcast" style="width: 100%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

ส่วนนี้ใช้สร้างข้อความ Broadcast ที่ต้องการส่ง โดยระบบรองรับการเพิ่มข้อความได้หลายรายการตามจำนวนสูงสุดที่กำหนด

* **สร้างข้อความ Broadcast:** กล่องหลักสำหรับสร้างข้อความที่จะส่ง
* **ข้อความ:** แสดงจำนวนข้อความที่สามารถเพิ่มได้ เช่น สูงสุด 5 ข้อความต่อครั้ง
* **ข้อความ #1:** แสดงลำดับของข้อความในชุด Broadcast
* **ประเภทข้อความ:** ใช้เลือกชนิดของข้อความที่ต้องการส่ง
* **ข้อความ:** ช่องสำหรับพิมพ์เนื้อหาที่ต้องการ Broadcast
* **เพิ่มข้อความ:** ใช้เพิ่มข้อความรายการถัดไปในชุด Broadcast
* **ดูตัวอย่างและส่ง:** ใช้ตรวจสอบตัวอย่างข้อความก่อนเข้าสู่ขั้นตอนยืนยันส่ง

จากภาพตัวอย่าง ระบบแสดงสถานะ **ไม่สมบูรณ์** เนื่องจากยังไม่ได้กรอกข้อความในช่อง **ข้อความ** จึงยังไม่ควรส่ง Broadcast

> ข้อควรระวัง: หากข้อความยังไม่ครบถ้วน ระบบอาจแสดงสถานะไม่สมบูรณ์ และไม่ควรกดส่งจนกว่าจะกรอกข้อมูลครบ

### 3. ประเภทข้อความที่รองรับ

<a href="/images/broadcast/image2.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/broadcast/image2.png" alt="ประเภทข้อความที่รองรับใน Broadcast" style="width: 100%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

ส่วนนี้ใช้เลือกประเภทข้อความที่ต้องการส่งผ่าน Broadcast โดยระบบรองรับหลายรูปแบบ เพื่อให้แอดมินเลือกให้เหมาะกับเนื้อหาที่ต้องการสื่อสาร

ประเภทข้อความที่แสดงในหน้าจอ ได้แก่

* **ข้อความ:** ใช้ส่งข้อความตัวอักษรธรรมดาผ่าน LINE
* **สติกเกอร์:** ใช้ส่งสติกเกอร์ LINE
* **รูปภาพ:** ใช้ส่งรูปภาพไปยังห้องแชท
* **วิดีโอ:** ใช้ส่งไฟล์วิดีโอ
* **เสียง:** ใช้ส่งไฟล์เสียง
* **ไฟล์:** ใช้ส่งไฟล์แนบ
* **Flex Message:** ใช้ส่งข้อความรูปแบบการ์ดหรือเทมเพลตที่ออกแบบไว้

> ข้อควรทราบ: ควรเลือกประเภทข้อความให้เหมาะกับเนื้อหาที่ต้องการส่ง หากเป็นประกาศสั้น ๆ ควรใช้ **ข้อความ** แต่หากต้องการรูปแบบสวยงามหรือมีปุ่มกด ควรใช้ **Flex Message**

### 4. หน้าต่างยืนยันการ Broadcast

<a href="/images/broadcast/image1.png" target="_blank" style="display: block; margin: 20px 0;">
  <img src="/images/broadcast/image1.png" alt="หน้าต่างยืนยันการ Broadcast" style="width: 100%; max-width: 800px; border-radius: 8px; box-shadow: 0 2px 6px rgba(0,0,0,0.15);">
</a>

หน้าต่างนี้จะแสดงหลังจากกด **ดูตัวอย่างและส่ง** ใช้สำหรับตรวจสอบข้อมูลก่อนยืนยันส่ง Broadcast จริง

ข้อมูลที่แสดงในหน้าต่างประกอบด้วย

* **ยืนยันการ Broadcast:** หัวข้อหน้าต่างสำหรับยืนยันการส่งข้อความ
* **จำนวนผู้ที่รับ Broadcast ได้:** แสดงจำนวนผู้รับที่ระบบตรวจสอบได้ในขณะนั้น
* **ตรวจจำนวนผู้รับอีกครั้ง:** ใช้ตรวจสอบจำนวนผู้รับ Broadcast ใหม่ก่อนส่ง
* **ตัวอย่างข้อความที่จะส่ง:** แสดงตัวอย่างข้อความทั้งหมดที่จะถูกส่ง
* **ยกเลิก:** ปิดหน้าต่างโดยไม่ส่ง Broadcast
* **ยืนยันส่ง Broadcast:** ใช้ยืนยันและส่งข้อความ Broadcast จริง

> ข้อควรระวัง: ก่อนกด **ยืนยันส่ง Broadcast** ควรตรวจสอบตัวอย่างข้อความและจำนวนผู้รับอีกครั้ง เพราะหลังจากยืนยันแล้วข้อความจะถูกส่งออกไปยังผู้รับตามเงื่อนไขของระบบ

## ปัญหาที่พบบ่อย (FAQ)

**1. ทำไมปุ่มส่ง Broadcast ใช้งานไม่ได้ ?**

* **สาเหตุ:** ข้อความอาจยังไม่สมบูรณ์ หรือยังไม่ได้กรอกเนื้อหาที่จำเป็น
* **วิธีแก้ไข:** ตรวจสอบสถานะของข้อความ และกรอกข้อมูลในช่อง **ข้อความ** หรือข้อมูลที่จำเป็นให้ครบ

**2. ทำไมจำนวนผู้รับ Broadcast เป็น 0 ?**

* **สาเหตุ:** อาจยังไม่มีห้องแชทที่เข้าเงื่อนไข หรือระบบยังไม่ได้ตรวจสอบจำนวนผู้รับล่าสุด
* **วิธีแก้ไข:** กด **ตรวจจำนวนผู้รับอีกครั้ง** และตรวจสอบว่ามีห้องแชท LINE ที่เปิดอยู่ในระบบ

**3. ทำไมส่ง Broadcast แล้วผู้รับไม่ครบตามที่คาดไว้ ?**

* **สาเหตุ:** จำนวนผู้รับจริงอาจขึ้นอยู่กับสถานะห้องแชท การบล็อกบัญชี หรือข้อจำกัดของ LINE Official Account
* **วิธีแก้ไข:** ตรวจสอบจำนวนผู้รับก่อนส่ง และตรวจสอบสถานะของห้องแชท LINE ที่เกี่ยวข้อง

**4. ทำไมเลือกประเภทข้อความบางแบบแล้วส่งไม่ได้ ?**

* **สาเหตุ:** ประเภทข้อความบางแบบอาจต้องมีข้อมูลเพิ่มเติม เช่น ไฟล์ รูปภาพ วิดีโอ หรือ Flex Message
* **วิธีแก้ไข:** ตรวจสอบว่ากรอกหรือแนบข้อมูลที่จำเป็นครบแล้วตามประเภทข้อความที่เลือก

**5. ทำไมข้อความที่ส่งไปไม่ตรงกับที่ต้องการ ?**

* **สาเหตุ:** อาจไม่ได้ตรวจสอบตัวอย่างข้อความก่อนกดยืนยันส่ง
* **วิธีแก้ไข:** ใช้ปุ่ม **ดูตัวอย่างและส่ง** เพื่อตรวจสอบตัวอย่างทุกครั้งก่อนกด **ยืนยันส่ง Broadcast**
